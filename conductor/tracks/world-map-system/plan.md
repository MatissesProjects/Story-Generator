# World Map System Implementation Plan (Detailed)

## Phase 1: Geographical Foundation
- [x] **Database Expansion**: Update `schema.sql` with `locations`, `regions`, `paths`, and `entity_positions`.
- [x] **Location Manager**: Create `world_engine.py` to handle CRUD for locations and coordinates.
- [x] **Spatial Logic**: Implement a "Coordinate Solver" that places new locations near existing ones based on LLM-described directions.

## Phase 2: The Cartographer Agent
- [ ] **Discovery Loop**: Integrate `world_engine` into `director.identify_location()`.
- [ ] **Map Metadata**: Create a prompt that extracts "Geographical Neighbors" from story text.
- [ ] **Path Generation**: Automatically create `Paths` between locations as the player travels.

## Phase 3: Visual Map Canvas
- [ ] **Frontend Viewer**: Create a `static/map.html` or a modal component in `app.js` using HTML5 Canvas.
- [ ] **Icon System**: Represent characters and landmarks with unique icons (optionally generated by `vision.py`).
- [ ] **Tile Set**: Generate a base set of biome tiles (Forest, Mountain, Plain, City) to serve as map backgrounds.

## Phase 4: Movement & Encounters
- [ ] **Travel State**: Update `server.py` to handle "In Transit" states.
- [ ] **Encounter Trigger**: Implement logic for the Director to inject "Road Events" based on path type.
